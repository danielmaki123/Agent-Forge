# Tasks

Task definitions with scope, acceptance criteria, and verification commands.

Create tasks using `templates/task.md` or the `/new-task` command.

Task naming: `TASK-XXX-<descriptive-slug>.md`

Rules:
- One task = one reviewable unit of work
- Every task must have executable acceptance criteria
- Every task must reference the contracts it consumes
- Tasks are executed sequentially unless `/autopilot` is used
