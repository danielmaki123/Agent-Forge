Filosofía Central

📋

Especificación antes que código

Los contratos definen el comportamiento. El código implementa contratos. Nunca al revés.

Codificar sin specs → +41% complejidad de código
🛡️

Autonomía gobernada

Los agentes operan libremente dentro de límites. Se detienen ante el riesgo, no después del daño.

Gobernanza solo con prompts → 26.67% tasa de violación
✅

Verificar, no confiar

Cada tarea tiene verificación ejecutable. "Terminado" significa que los checks pasan.

22.7% de problemas de IA persisten en las últimas versiones
⚡

Primero enviar, después gobernar

TASK-001 se ejecuta antes de agregar más gobernanza. Sin parálisis por análisis.

Máximo 2 ADRs antes de completar la primera tarea

3 Capas de Gobernanza

De sugerencias a enforcement. Cada capa reduce las violaciones.

1

Instrucciones Markdown

~26-67% violaciones

AGENTS.md — constitución de agentes legible por humanos con roles, flujo y reglas de seguridad.

AGENTS.md README.md
↓ más fuerte ↓
2

Políticas YAML

~5-10% violaciones

Reglas parseables por máquina. CI/CD valida commits. Límites de archivos, gates de seguridad, requisitos de calidad.

policies/boundaries.yaml policies/safety.yaml policies/quality.yaml
↓ la más fuerte ↓
3

Quality Gates

~0% violaciones

Kill switches numéricos en PROJECT.yaml. Límites duros: máx 5 archivos, 100 líneas, 15 min, 3 reintentos.

PROJECT.yaml → quality_gates PROJECT.yaml → checks

Kill Switches y Seguridad

Investigación: agente de 10 pasos a 95% de precisión → solo 59.9% de éxito del sistema.

🔴 Prohibido Siempre

  • Deploy a producción
  • Commitear secretos
  • Migrations sin aprobación
  • Eliminar tests para pasar checks
  • Borrar datos de producción

🟡 Requiere Aprobación

  • Agregar dependencias
  • Modificar esquema de BD
  • Cambiar arquitectura
  • Escrituras a servicios externos
  • Modificar auth/seguridad

⛔ Auto-Stop

  • Tarea excede 5 archivos
  • Tarea excede 100 líneas
  • Agente corriendo >15 minutos
  • 3 fallos consecutivos
  • Incertidumbre sobre impacto

Comandos

Atajos operativos. Escribe el comando, el agente ejecuta el protocolo.

/continue-roadmap

Ejecutar la siguiente tarea pendiente

/autopilot

Ejecutar hasta 3 tareas con compactación de contexto

/worker-decision

Decidir quién ejecuta + plan de implementación

/review

Revisar tarea contra contratos + pre-mortem

/red-team

Revisión adversarial — encontrar problemas, no confirmar calidad

/status

Reconstruir estado desde el repo, no desde memoria del chat

/checks

Ejecutar todas las verificaciones de PROJECT.yaml

/stop

Parada de emergencia con evaluación de riesgo

/resolve-blocker

Analizar bloqueo sin implementar código

/new-task

Crear tarea estructurada desde template

/new-adr

Crear registro de decisión arquitectónica
